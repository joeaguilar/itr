**Root cause (confirmed, code in hand):** urgency snapshots are sub-second
wall-clock dependent. `age = config.age * clamp(days_since(created_at)/10,0,1)`;
`days_since` (src/util.rs:133) uses `Utc::now()` in real seconds; compact renderer
drops zero components (`.filter(|(_,v)| *v != 0.0)`). Same-second seed→read ⇒ age
exactly 0.0 (omitted); ≥1s ⇒ age≈4.6e-06 (shown as `age=0.0`, and shifts the JSON
urgency total). `src/urgency.rs` unchanged this session — latent gap exposed, not
caused, by the new urgency-bearing snapshots. #139 covered stats/graph JSON only.
Blast radius: 15 compact + 37 json snapshots carry urgency.

**Orchestrator process errors (owned):** (1) first wave-gate wrapper piped
`integration.sh | tail`, so `$?` read `tail` (always 0) → briefly reported the
gate green when red; (2) then confabulated a wrong cause ("age=2.0, ~10 days old")
before clean diagnostics returned. Both corrected; true cause is the age race above.
